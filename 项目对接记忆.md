## 🎯 项目是什么

**真心话大冒险炫酷转盘** —— 朋友聚会用的双层大转盘小工具：

1. **玩家页 `/`**：先转主转盘抽「真心话 / 大冒险」，再自动续转对应大转盘抽具体题目。全屏弹窗显示，手机端适配。
2. **控制台 `/admin.html`**：管理题库、调权重、改设置、密码保护。

**技术特点**：
- Node.js 原生 `http` 模块，**零依赖**（package.json 没 dependencies）
- 数据持久化：单文件 JSON（`data/wheel-data.json`）
- 没有数据库、没有 Express、没有任何中间件框架

## 🏗️ 架构关键点（不要打破）

### 1. server.js 的双写入路径（极重要）

服务器有**两条写入路径**，对应两种调用者：

| 路径 | 调用者 | 鉴权 | 能改什么 |
|---|---|---|---|
| `POST /api/data` → `writePlayerSafe()` | 玩家页 wheel.js | ❌ 不需要 | **白名单**：仅 `settings.autoSpinNext` 和 `settings.peopleCount` |
| `POST /api/admin/save` → `writeDataInternal()` | 控制台 admin.js | ✅ 需要 X-Admin-Auth | main / truth / dare / settings 全部（除密码外） |

**为什么这样分**：玩家页会调 `apiSaveData()` 来保存抽完后的人数偏好等（见 `wheel.js:563, 590`），如果加上密码保护就会把玩家挡在外面。所以玩家页用一个**字段白名单受限的开放接口**，管理员用一个**鉴权但能改全部的接口**。

**白名单实现**（server.js 中 `writePlayerSafe`）：
```js
const allowedKeys = ['autoSpinNext', 'peopleCount'];
const mergedSettings = { ...existing.settings };
for (const k of allowedKeys) {
  if (k in incomingSettings) mergedSettings[k] = incomingSettings[k];
}
```
玩家就算用 fetch 调 `/api/data` 想 POST `truth: ['HACKED']` 也不会生效。

### 2. 密码绝不通过 GET 暴露

`readDataInternal()` 读完整数据（含 `adminPassword`）；`readDataPublic()` 是它的过滤版，永远剥离 `settings.adminPassword`。**所有 GET `/api/data` 走 readDataPublic，POST 写完后返回也走它**。前端任何时候 fetch `/api/data` 都拿不到密码。

### 3. 密码只能通过专用接口改

- `writeDataInternal()` 不管传进来什么 newData，都从磁盘读现有密码 force preserve（`normalized.settings.adminPassword = existing.settings.adminPassword || ''`）
- 唯一改密码的入口是 `POST /api/admin/password`（需鉴权）→ 调 `setAdminPassword()`
- 这样导入 / 改权重 / 重置都不会意外清空密码

### 4. Token = 密码本身（简化方案）

没用 JWT 也没用 session 表。`/api/admin/login` 校验密码后直接把**密码本身**当 token 返回给前端。前端存 sessionStorage（`tdw_admin_token`），后续请求带 `X-Admin-Auth: <密码>` 头。服务器收到后跟磁盘上的密码字符串比对。

**理由**：项目体量小、内网 / 朋友聚会场景、零依赖。如果以后要上公网，记得：
- 加 HTTPS（反向代理层做）
- 改用 hash + salt 存密码（如 bcrypt）
- token 改成随机字符串 + 过期时间

### 5. Login overlay 在 admin.html 里，不要单独的 login.html

`/admin.html` 加载时 `bootstrap()` → `apiAdminStatus()` 判断是否需要密码：
- 不需要 → 直接渲染
- 需要 + sessionStorage 有 token + token 还有效（用 `/api/admin/login` 复验）→ 直接渲染
- 否则 → 显示 `#loginOverlay` 遮罩（fixed inset:0），输入正确后隐藏

仓库里那个 `public/login.html` 是用户之前自己做的，跟这套机制冲突，**用户自己会删**。

### 6. 题目去重逻辑（容易踩坑）

`normalizeList()` 用 Map 按文本去重，重复保留较大权重：
```js
if (!seen.has(text) || seen.get(text) < weight) seen.set(text, weight);
```

**意味着**：内联编辑时如果改成跟另一行相同的文本，保存后会被合并。所以 admin.js 的 `commitTextEdit()` 在保存前**前端先做重复检测**，重复就回滚 + 提示。

### 7. 内联编辑用 contenteditable + blur/Enter 提交

不是 input 框 —— 而是 `<span contenteditable="true">`。这样不会破坏原有的 `<em>#1</em>` 序号显示。

- Enter → 阻止默认 + 触发 blur → 提交
- Esc → 恢复 dataset.original → blur
- paste → 阻止默认，强制纯文本插入（避免粘贴富文本污染）
- 多行换行折成空格：`newText.replace(/\s+/g, ' ').trim()`

## ⚠️ 已知避坑

### A. server.js 必须在【根目录】，不在 public/

代码里 `PUBLIC_DIR = path.join(__dirname, 'public')`，意味着 server.js 必须放在 public/ 的**父目录**。
**用户最初上传的 zip 把 server.js 错放在了 public/ 里**，仓库里现在两个位置都有同名文件（内容一样），但 PM2 启动的是根目录那个。`public/server.js` 是冗余，可删。

### B. `data/wheel-data.json` 永远别覆盖

题库和密码全在这。每次部署 / 推送都要确认这个文件不会被动到。`.gitignore` 应该已经排除（仓库现在没 .gitignore，**TODO**：未来加一个）。

### C. ~~GM 飘屏（broadcast）功能是半成品~~ ✅ v2.3.0 已修复

历史遗留：原版 server.js 没实现 `POST /api/broadcast`，admin 点"发送飘屏"按钮时前端会 `Unexpected token '<'` 报错（因为接口走到 `serveStatic` 返回了 HTML）。

**v2.3.0 修复**：
- server.js 加 `POST /api/broadcast`（鉴权,body `{text}`,空内容 400）
- 写入 `data.broadcast = { id: Date.now(), text, createdAt }`
- `defaultData` / `normalizeData` / `writePlayerSafe` 全都保留 broadcast 字段
- admin.js 的 `apiSendBroadcast` 改用 `adminHeaders()` 带 `X-Admin-Auth`
- 玩家页轮询 GET `/api/data` 时拿到新 `broadcast.id` 就触发飘屏（wheel.js 原有逻辑无需改）

### D. 玩家页 `peopleCount` 防覆盖

`wheel.js:88` 用了 `document.activeElement !== peopleCountInput` 判断 —— 玩家正在输入时不要被 8 秒一次的轮询数据回写覆盖。改逻辑时别破坏这个。

### E. 浏览器缓存

style.css / 各种 .js 文件，服务端给的是 `Cache-Control: public, max-age=3600`（见 `serveStatic`）。改完前端代码用户报问题时，第一句先让他 Ctrl+F5。

## 🔌 API 完整清单

| 路径 | 方法 | 鉴权 | 说明 |
|---|---|---|---|
| `/api/data` | GET | ❌ | 读全部数据（自动剥离 adminPassword） |
| `/api/data` | POST | ❌ | 玩家保存（白名单：仅 autoSpinNext / peopleCount） |
| `/api/admin/status` | GET | ❌ | `{ ok, passwordRequired }` |
| `/api/admin/login` | POST | ❌ | body `{password}`，对了返回 `{ ok, token, passwordRequired }`，错返 401 |
| `/api/admin/save` | POST | ✅ | body `{data}`，全量保存（密码字段被 force preserve） |
| `/api/admin/password` | POST | ✅ | body `{newPassword}`，留空字符串=关闭密码保护 |
| `/api/reset` | POST | ✅ | 恢复示例数据（密码保留） |
| `/api/broadcast` | POST | ✅ | body `{text}`(≤200 字)，写入 `data.broadcast`，玩家页轮询触发飘屏 |

鉴权方式：HTTP 头 `X-Admin-Auth: <密码本身>`。
